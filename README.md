A community-curated knowledge base of atomic developer tips, delivered through a browser-extension new-tab page and a public site at **[tipbytab.moidris.com](https://tipbytab.moidris.com/)**. Each new tab shows one bite-sized tip — code first, short explanation, and the contributor's GitHub handle. Your handle renders in every reader's new tab: that recognition loop is the engine, not a side feature.

- **Zero backend.** GitHub is the platform: tips are Markdown PRs, moderation is PR review, attribution is commit history, the CDN is jsDelivr. Cost $0.
- **No telemetry, no tracking, no accounts.** Every tip is human-authored and human-reviewed.
- **Respects your new tab.** `Esc` blanks the page instantly — no hijacking.

Stacks are open-ended — one directory per stack under `tips/` (currently `go`, `laravel`, `nest`, `next`, `nuxt`, `php`, `react`, `rust`, `vue`). Adding a stack = adding a directory.

## How it works

```
contributor PR → CI validate → maintainer merge → CI assigns id + builds dist/
  → jsDelivr serves → extension fetches
  → Cloudflare Pages serves → site renders tip at dailydevtip.dev/<stack>/<slug>
```

Three surfaces, one Markdown source:

| Surface | Role |
|---------|------|
| `tips/` (this repo) | Markdown source. One file per tip. PR = submission, CI = validation + ID assignment. |
| `dist/` (this repo) | JSON build output served via jsDelivr to the extension. Auto-generated by CI on merge. |
| `site/` (this repo) | Astro static site. Reads `tips/` at build time → HTML at `dailydevtip.dev`. Deployed to Cloudflare Pages. |
| `daily-dev-tip-extension` | WXT + TypeScript MV3 new-tab page. Fetches `dist/*.json` from jsDelivr, falls back to bundled `fallback.json`. |

## Local development

Package manager is **bun**.

### Content pipeline

```bash
bun install
bun run validate      # schema + content rules + dup check across all tips
bun run assign-id     # CI-time: fill ids for new tips
bun run build         # generate dist/
```

Add a tip: create `tips/<stack>/<kebab-title>.md` — frontmatter, then a code block (≤ 25 lines) first, then prose (≤ 600 chars). `id` and `codeHash` are assigned by CI, never by you. See [CONTRIBUTING.md](CONTRIBUTING.md).

### Public site (`site/`)

The site reads tips directly from `tips/` via a symlink (`site/src/content/tips → ../../tips`). No sync step needed.

```bash
cd site
bun install
bun run dev           # dev server at http://localhost:4321
bun run build         # static build → site/dist/
bun run preview       # preview the production build locally
```

Pages generated at build time:
- `/` — landing page
- `/<stack>/` — stack archive (Laravel, Vue, Nest)
- `/<stack>/<slug>/` — individual tip with author card + install CTA
- `/contributors/` — all contributors sorted by tip count
- `/contributors/<handle>/` — contributor profile + all their tips
- `/contribute/` — editorial bar and submission guide
- `/rss.xml`, `/<stack>/rss.xml` — RSS feeds
- `/sitemap-index.xml` — generated sitemap

### Extension repo

```bash
cd ../daily-dev-tip-extension
bun install
bunx wxt prepare      # run once after install (bun blocks the postinstall)
bun run dev           # loads into Chrome
bun run test          # vitest
bun run compile       # tsc --noEmit typecheck
bun run build         # production MV3 build → .output/chrome-mv3
```

### Loading tips into the extension

The extension never bundles tips — it fetches JSON at runtime. `BASE` in the extension's `lib/tip-service.ts` picks the source per build mode:

| Build | Tips source |
|---|---|
| `bun run dev` | `http://localhost:8787/dist` |
| `bun run build` (prod) | `https://cdn.jsdelivr.net/gh/hnooz/tip-by-tab@main/dist` |
| fetch fails | bundled `assets/fallback.json` |

**Local loop** — serve this repo's root (not `dist/` itself; `BASE` includes the `/dist` prefix) with CORS enabled, since the new-tab page fetches cross-origin without a localhost host permission:

```bash
# terminal 1 — this repo
bun run build
bunx serve . -l 8787 --cors

# terminal 2 — extension repo
cd ../daily-dev-tip-extension
bun run dev
```

Verify: `curl http://localhost:8787/dist/rotation.json` returns JSON, and the new-tab page's DevTools Network shows requests to `localhost:8787/dist/`. A console warning "CDN fetch failed, using bundled fallback" means the server is down or CORS is missing.

**Production** — push `main` (with the CI-built `dist/`) to GitHub; jsDelivr serves it with zero setup. Note: jsDelivr caches `@main` refs for ~12 h, so freshly merged tips reach readers with a delay.

## Deploy

**Content pipeline** — CI runs on every PR (`validate.yml`) and on every merge to `main` (`build.yml`). The built `dist/` is committed back and served via jsDelivr automatically.

**Site** — `deploy-site.yml` triggers on push to `main` when `tips/**` or `site/**` changes. Requires two GitHub secrets: `CF_API_TOKEN` and `CF_ACCOUNT_ID` (Cloudflare Pages).

## Contributing

Tips go through PR review against the editorial bar in [CONTRIBUTING.md](CONTRIBUTING.md). One tip per file, no duplicates, `author.github` required. Every merged tip gets a permanent URL at `dailydevtip.dev/<stack>/<slug>`.
